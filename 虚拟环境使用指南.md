# 虚拟环境使用指南

## 🚀 快速开始

### 1. 首次设置（只需运行一次）
```bash
# 双击运行或在命令行执行：
setup_env.bat
```

这会：
- ✅ 检查Python环境
- ✅ 创建虚拟环境 `venv/`
- ✅ 安装所需依赖
- ✅ 验证环境配置

### 2. 日常使用
```bash
# 双击运行：
启动翻译环境.bat
```

这会：
- ✅ 激活虚拟环境
- ✅ 检查项目状态
- ✅ 启动翻译脚本

## 📁 文件说明

- **setup_env.bat** - 首次环境设置脚本
- **启动翻译环境.bat** - 日常使用启动脚本  
- **requirements.txt** - Python依赖列表
- **venv/** - 虚拟环境目录（自动创建）

## 🔧 手动使用（高级用户）

### 激活虚拟环境
```bash
# Windows
venv\Scripts\activate.bat

# 或
venv\Scripts\activate
```

### 安装依赖
```bash
pip install -r requirements.txt
```

### 运行翻译脚本
```bash
python ai_translate_simple.py
```

### 退出虚拟环境
```bash
deactivate
```

## ✨ 虚拟环境优势

1. **环境隔离**: 不影响系统Python环境
2. **依赖管理**: 精确控制包版本
3. **可移植性**: 易于在不同机器上复现
4. **清理简单**: 删除venv文件夹即可

## 📊 项目状态检查

脚本会自动显示：
- 总notebook文件数量
- 已翻译文件数量
- 待翻译文件数量

## ⚠️ 注意事项

1. **首次运行**: 必须先执行 `setup_env.bat`
2. **API配置**: 仍需在 `ai_translate_simple.py` 中配置API密钥
3. **网络连接**: 确保能访问API服务
4. **磁盘空间**: 虚拟环境需要约50MB空间

## 🛠️ 故障排除

### 问题：虚拟环境创建失败
**解决**: 确保Python版本3.8+，检查磁盘空间

### 问题：激活失败  
**解决**: 检查venv目录是否存在，重新运行setup_env.bat

### 问题：依赖安装失败
**解决**: 检查网络连接，尝试手动运行：
```bash
venv\Scripts\activate.bat
pip install requests
```

---

## 📋 完整工作流程

1. **首次设置**: 运行 `setup_env.bat`
2. **配置API**: 编辑 `ai_translate_simple.py` 中的API密钥
3. **开始翻译**: 运行 `启动翻译环境.bat`
4. **查看结果**: 翻译文件保存在 `notebooks-zh/` 目录
